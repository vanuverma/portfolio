Overview

A standards-based verifiable credential platform implementing W3C Verifiable Credentials specification. The system enables users to download cryptographically secure credentials and present them to verifiers with selective disclosure capabilities, ensuring privacy while maintaining verification integrity.

System Architecture

Download Flow

  1. User Authentication: User logs in through Auth0 identity provider
  2. Identity Validation: Platform vendor validates user identity
  3. Data Retrieval: Digital License Platform retrieves user's licence data
  4. Credential Preparation: License Issuer prepares verifiable credential according to W3C standards
  5. PKI Encryption: PKI vendor encrypts the credential for secure transmission
  6. Mobile Delivery: Encrypted verifiable credential is delivered to Service NSW Mobile App
  7. Secure Storage: Credential is securely stored on the mobile device

Verification Flow

  1. Verifier Registration: Verifier system registers and validates with the platform
  2. Request Generation: Platform vendor prepares verification request with required attributes
  3. QR Code Creation: Digital License Platform generates verification request link
  4. Verifier Display: Verification link is shown as QR code on verifier website
  5. User Scanning: User scans verification QR code using Service NSW Mobile App
  6. Selective Disclosure: User selects which credential attributes to share
  7. Verification Link: User verifies and shares selected data with verifier
  8. Cryptographic Validation: Platform performs cryptographic verification of shared credential
  9. Result Delivery: Verification response is sent to verifier system
  10. Confirmation: Verifier receives verification result on their website
