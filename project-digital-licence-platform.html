System Architecture

Download Flow

  1. User Authentication: Users login through Auth0 identity provider
  2. Identity Validation: Platform vendor validates user credentials
  3. Data Retrieval: Kubernetes-based platform retrieves licence data from backend systems
  4. Licence Issuance: License Issuer service prepares the digital licence with PKI encryption
  5. Notification Storage: License update notifications are stored in MongoDB database
  6. Mobile Delivery: Encrypted licence data is delivered to Service NSW Mobile App

Verification Flow

  1. User Authentication: User logs in via Auth0
  2. QR Code Generation: Digital licence is presented as a QR code on mobile app
  3. Verifier Scanning: External verifier scans the QR code
  4. Cryptographic Validation: Platform performs PKI-based cryptographic verification
  5. Response Delivery: Kubernetes cluster processes and returns verification response
  6. Identity Confirmation: Validation provider confirms licence authenticity
